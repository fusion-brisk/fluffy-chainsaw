# Autonomous phased refactor

Выполни рефакторинг по плану автономно, с проверкой после каждой фазы.

## Входные данные

Пользователь предоставит план (inline или ссылку на `.claude/specs/`).
Если плана нет — создай его в `.claude/specs/in-progress/` и покажи для подтверждения.

## Workflow

Для каждой фазы плана:

1. **Объяви фазу** — коротко: "Фаза N: <название>"
2. **Реализуй** — внеси изменения
3. **Проверь** — запусти `npm run verify` (typecheck + lint + test + build)
   - Если падает — исправь автоматически (до 3 попыток)
   - Если не удаётся за 3 попытки — **СТОП**, покажи ошибку, спроси
4. **Закоммить** — формат: `refactor: <фаза> — <описание>`
5. **Перейди к следующей фазе**

## Правила

- После КАЖДОЙ фазы обязателен зелёный `npm run verify`
- Не пропускай фазы и не объединяй их
- Каждая фаза = отдельный коммит
- Если фаза затрагивает >10 файлов, разбей на подфазы
- После всех фаз: `git log --oneline` + итоговый отчёт (файлы, строки, фазы)
- Если пользователь скажет "стоп" — немедленно прекрати

## Ограничение

Если тест-покрытие затронутых модулей <50%, предупреди пользователя перед началом.
Проверь: `npm run test -- --reporter=verbose 2>&1 | grep -c "✓"` vs количество изменяемых функций.
